# 飞书 Bot 配置指南

本文档介绍如何创建飞书自建应用机器人，并配置 TradingAgent 系统通过该机器人**私聊推送**交易信号给你。

系统采用**自动配对**机制：你只需在飞书中给 Bot 发一条消息，即完成绑定，无需手动查询或配置 `open_id`。

---

## 配置步骤

### 第 1 步：创建飞书自建应用

1. 打开 [飞书开放平台](https://open.feishu.cn/app)，用你的飞书账号登录
2. 点击 **「创建企业自建应用」**
3. 填写应用信息：
   - **应用名称**：`TradingAgent`（或任意名称）
   - **应用描述**：`A 股量化交易信号推送`
   - **应用图标**：上传一个图标（必填）
4. 点击 **「创建」**

### 第 2 步：获取 App ID 和 App Secret

1. 进入刚创建的应用详情页
2. 左侧菜单点击 **「凭证与基础信息」**
3. 页面中可以看到：
   - **App ID**：`cli_xxxxxxxxxxxxxxxx`
   - **App Secret**：点击「显示」后复制

> ⚠️ **App Secret 是敏感信息**，请勿泄露或提交到代码仓库。

### 第 3 步：开启机器人能力

1. 左侧菜单点击 **「应用能力」→「添加应用能力」**
2. 找到 **「机器人」** 能力，点击 **「添加」**

### 第 4 步：配置权限

1. 左侧菜单点击 **「权限管理」**
2. 搜索并开通以下权限：

| 权限 | 权限标识 | 用途 |
|------|---------|------|
| 获取与发送单聊、群组消息 | `im:message` | 发送信号推送消息 |
| 以应用的身份发消息 | `im:message:send_as_bot` | Bot 身份发送私聊消息 |
| 读取用户发给机器人的单聊消息 | `im:message.p2p_msg:readonly` | 接收用户私聊消息以完成自动配对 |

3. 点击 **「批量开通」**

### 第 5 步：配置事件订阅（长连接模式）

1. 左侧菜单点击 **「事件与回调」→「事件订阅」**
2. 接收方式选择 **「使用长连接接收事件」**（无需公网 IP）
3. 添加事件：搜索并订阅 **`im.message.receive_v1`**（接收消息）

> 长连接模式下，系统启动后 lark-oapi SDK 会自动维护 WebSocket 连接，接收用户发给 Bot 的私聊消息。

### 第 6 步：发布应用

1. 左侧菜单点击 **「版本管理与发布」**
2. 点击 **「创建版本」**，填写版本号（如 `1.0.0`）和更新说明
3. 点击 **「提交发布」**
4. 等待企业管理员审批（如果你是管理员，可以直接在管理后台审批通过）

> 应用发布后才能在飞书客户端中使用。

### 第 7 步：配置 TradingAgent

将获取到的信息填入 `.env` 文件：

```bash
# 飞书 Bot 配置（仅需 App ID 和 Secret，open_id 通过自动配对获取）
FEISHU_APP_ID=cli_xxxxxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 第 8 步：绑定推送（自动配对）

1. 启动 TradingAgent 系统：`docker compose up`
2. 在飞书客户端中搜索你的 Bot 应用名（如 `TradingAgent`）
3. 给 Bot 发送任意消息（如发送 `绑定`）
4. Bot 自动回复：**✅ 绑定成功，后续交易信号将私聊推送给你**
5. 完成！后续所有交易信号将自动私聊推送给你

> 系统通过 `im.message.receive_v1` 事件自动获取你的 `open_id` 并持久化到 SQLite，无需手动查询或配置。

### 第 9 步：验证推送

通过 CLI 手动触发一次扫描来验证：

```bash
trading-agent run scan
```

如果配置正确，你会在飞书中收到 Bot 的私聊信号推送消息。

## 常见问题

**Q: 发消息失败提示 "permission denied"**

检查第 4 步中的权限是否已全部开通，并且应用版本已发布。

**Q: Bot 没有自动回复"绑定成功"**

确认第 5 步中事件订阅已正确配置（`im.message.receive_v1`，长连接模式），并且系统正在运行中。

**Q: App Secret 泄露了怎么办？**

在飞书开放平台应用详情页 → 凭证与基础信息 → 点击「重置」生成新的 Secret，然后更新 `.env` 文件并重启服务。

**Q: 可以同时推送给多个人吗？**

MVP 仅支持推送给一个用户（最后一个完成配对的用户）。如需多人推送，后续可扩展为支持多用户绑定。

**Q: 如何解除绑定？**

删除 SQLite 中 `feishu_bindings` 表的记录，重启系统后重新私聊 Bot 即可重新绑定。
