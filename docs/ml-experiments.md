## 系统配置

### 因子引擎

- **Alpha158Engine**: 191 个因子（159 技术 + 32 基本面）
- 技术因子: KBar(9) + Price(5) + Rolling(145, 29 指标 × 5 窗口 [5,10,20,30,60])
- 基本面因子: PE/PB/PS/PCF/ROE/营收增速/利润增速/毛利率/市值/换手率等 32 个
- 价格使用前复权（`close * adj_factor`），确保跨除权日连续性

### 标签计算

```
label = (stock_fwd_adj_close[t+h] / stock_fwd_adj_close[t] - 1)
      - (benchmark_close[t+h] / benchmark_close[t] - 1)
```

- 默认 horizon h=5 交易日
- Benchmark: 沪深300（000300）
- 标签 winsorize 到 [-0.2, +0.2]
- 使用前复权价格计算，捕获分红收益

### Walk-Forward 训练框架

- 默认配置: train=18 月, val=3 月, test=3 月, step=3 月
- 17 个滑动窗口（2020-01 起，至 2025-12）
- 训练集: 使用时间段内**所有历史成分股并集**（~1311 只），最大化样本量
- 验证/测试集: 使用**时点成分股快照**（~800 只），模拟真实交易池
- Purged CV: 训练集去除最后 `label_horizon` 天，防止标签泄露到验证集

### LightGBM 默认参数

```python
DEFAULT_PARAMS = {
    "objective": "mae",
    "metric": "mae",
    "num_leaves": 31,
    "learning_rate": 0.02,
    "n_estimators": 2000,
    "subsample": 0.8,
    "colsample_bytree": 0.7,
    "min_child_samples": 100,
    "random_state": 42,
}
EARLY_STOPPING_ROUNDS = 200
MIN_ITERATIONS = 50  # 防止病态早停
```

### 回测引擎

- 每周调仓（ISO 周第一个交易日）
- 等权配置，默认 top_n=30, n_drop=10（TopkDropout）
- 交易成本: 印花税 0.1% + 佣金 0.03% + 滑点 0.1%
- 排除科创板（688/689 前缀）
- 5 步调仓: 分类 → 卖出 → 构建候选池 → 分配资金 → 结算

### 关键观察

1. **IC 与收益不直接对应**：W03 IC 最高（+0.078）但超额仅 -0.3%；W06 IC 为负（-0.025）但区间 Sharpe 最高（+3.26）——再次印证 IC-Sharpe 悖论在窗口粒度上同样成立。

2. **W16 是唯一负 RankIC 窗口**（-0.033），也是超额最差（-12.0%）——2025Q3 市场风格急转，STD30 主导的波动率模型失效。

3. **W01 负 IC**：首个窗口（2021Q4），训练数据仅从 2020-01 起（~21 个月），是所有窗口中训练样本最少的。

4. **风格切换明显**：W01-W12 以 **IMIN60 + PB** 主导（动量+价值风格），W13-W17 切换为 **STD30/STD20** 主导（波动率风格）——对应 2024 年 924 行情后 A 股市场结构性变化。

5. **树数分布**：10/17 窗口停在 MIN_ITER=50（早停触发），W06 是异常值（595 棵，唯一用满全部 191 因子）。树数更多不代表更好——W06 IC 为负但 Sharpe 最高，W05 仅 50 棵但 IC 第二高。

6. **因子使用率**：每窗口约用 145-175/191 个因子，约 20-46 个因子重要性为零。频繁为零的因子：IMIN5, KUP2, RANK10, RANK5（超短周期因子对 5d 预测无用）。

## Phase 0: 基础修复

> **注意**: Phase 0-1 的实验均在 **Top-10** 等权组合下进行（后来在 Phase 2 升级为 Top-30 + TopkDropout）。所有 Phase 0-1 结果均为 seed=42 单种子。

### EXP-00: MIN_ITERATIONS=50 早停守卫

| | |
|---|---|
| **Commit** | `34c18e2` |
| **问题** | 7/17 训练窗口仅训练 1-5 棵树，严重欠拟合。早停在噪声验证数据上过早触发。 |
| **方法** | 添加 `MIN_ITERATIONS=50` 守卫：如果早停最佳轮次 < 50，则以固定 50 棵树重新训练（无早停）。同时将 `EARLY_STOPPING_ROUNDS` 从 100 提升到 200。 |
| **原理** | 金融时序信号极弱（IC~0.03），验证集噪声大。50 棵树是 MAE 目标下的最低有效复杂度。 |

**结果：**

| 指标 | 修复前 | 修复后 |
|------|:------:|:------:|
| Mean IC | 0.033 | 0.037 |
| Mean RankIC | 0.058 | 0.063 |
| Sharpe | 0.160 | 0.308 |
| Total Return | +5.7% | +19.4% |

**结论：** 7/17 窗口升级到 50 棵树，是后续所有实验的基础。此后 13/17 窗口稳定停在 MIN_ITER=50，说明 50 棵树就是该信号强度下的自然平衡点。

| | |

### EXP-02: 标签截面 Z-Score 归一化（CSZScoreNorm）

| | |
|---|---|
| **Commit** | `524eba4` |
| **CLI** | `pangu train walkforward --normalize-label` |
| **问题** | Qlib 标准流程使用 CSZScoreNorm，是否能提升信号一致性？ |
| **方法** | 训练前对标签做截面 z-score 标准化：每天 label = (label - mean) / std。模型学习的是"相对其他股票的表现"而非"绝对超额收益"。 |
| **原理** | Z-score 消除不同截面天的绝对收益差异，理论上让模型专注于相对排名。 |

**结果：**

| 指标 | 基线 | Z-Score |
|------|:----:|:------:|
| RankIC | 0.063 | 0.063 |
| Sharpe | 0.308 | 0.045 |
| Total Return | +19.4% | -3.8% |

**失败分析：**
- 6/17 窗口出现欠拟合（z-score 压缩了梯度信号）
- 模型偏好"稳定高于均值"的低波动银行股（2025Q1 Top-10 中 7.8 只是银行）
- Z-score 标签本质上重新定义了损失函数，与 Top-N 绝对收益选股目标冲突

**结论：** Z-score 归一化仅适用于多空策略，对 Top-N 纯多头策略有害。Sharpe 0.045 远低于多种子基线最差值（0.346），无需多种子验证即可确认失败。该功能保留为可选（默认关闭）。

### EXP-03: LambdaRank 排序学习

| | |
|---|---|
| **Commit** | `adc8a92` |
| **CLI** | `pangu train walkforward --mode ranking --n-bins 10` |
| **问题** | 回归目标优化全截面 MAE，但我们只关心 Top-N 排名。LambdaRank 是否能直接优化头部排名质量？ |
| **方法** | 将连续标签离散化为 10 分位（0-9），使用 LGBMRanker + NDCG 损失函数训练。NDCG 对头部位置给予指数级更高权重。 |
| **原理** | Learning-to-Rank 理论上完美匹配 Top-N 选股：直接优化前 N 名排序精度。 |

**结果：**

| 配置 | Mean RankIC | Sharpe | Total Return | Max DD |
|------|:----------:|:------:|:------------:|:------:|
| Run 1 (lr=0.01) | -0.043 | -0.045 | -21.4% | -59.5% |
| Run 2 (lr=0.05, norm) | -0.048 | — | — | — |

**失败分析（根因）：**
1. NDCG 增益函数对最高分位指数加权 → 模型过拟合高波动动量股
2. A 股均值回归特性 → 这些股票样本外系统性跑输 → 负 IC
3. 800 只股票的 NDCG@k 梯度噪声极大（弱信号体制下）
4. MIN_ITER=50 固化了逆向排名模式
5. 73.5% 的交易日 Top-10 与回归基线零重叠

**结论：** LambdaRank 在弱信号体制（IC~0.03）下根本不可用。负 RankIC 远离任何合理噪声带，无需多种子验证。代码保留为选项但标记"请勿使用"。

### EXP-04: 早停变体（M1a / M1b）

| | |
|---|---|
| **Commit** | `89fec79`（RankIC 早停部分） |
| **CLI** | `pangu train walkforward --early-stop-metric rankic` |
| **问题** | MAE 早停是否过于保守？放宽早停或用 RankIC 作为早停指标是否能提升排名质量？ |
| **方法** | M1a: 将 MIN_ITER 从 50 提升到 200，允许更多树。M1b: 用自定义 RankIC（截面 Spearman 相关系数）替代 MAE 作为早停指标。 |

**Phase 1 首测（Top-10，无 TopkDropout）：**

| 配置 | Sharpe | vs 基线 (0.308) |
|------|:------:|:-------------:|
| 基线 | 0.308 | — |
| M1a (MIN_ITER=200) | 0.216 | -30% |
| M1b (RankIC 早停) | 0.207 | -33% |

**Phase 2 重测（Top-30 + TopkDropout, n_drop=10，seed=42 单种子）：**

| 配置 | RankIC | Sharpe | vs seed=42 基线 |
|------|:------:|:------:|:-------------:|
| 基线 (MAE, MIN_ITER=50) | 0.066 | 0.609 | — |
| M1a (MIN_ITER=200) | 0.069 | 0.589 | -3% |
| M1b (RankIC 早停) | 0.068 | 0.582 | -4% |
| M1a+M1b 组合 | 0.070 | 0.561 | -8% |

**关键发现：** TopkDropout 大幅缩小了 M1a/M1b 的劣势（-30%→-3%），说明 TopkDropout 能部分抵消模型不稳定性的影响。但基线仍然更优。

**失败分析：**
- M1ab 的模型指标更好（树数 CV 0.71 vs 1.56，截面标准差翻倍）
- 但选股稳定性更差（日重叠度 0.581 vs 0.620，排名相关 0.509 vs 0.545）
- 更多树 → 更复杂分裂 → 对因子微小变化更敏感 → 排名更不稳定 → 换手更高

**结论：** 首次发现 **IC-Sharpe 悖论**：提高模型预测质量（IC）反而降低投资组合表现（Sharpe）。50 棵树 MAE 早停是该信号强度下的最优隐性正则化。M1a/M1b 在 Phase 1 的 Sharpe（0.207~0.216）远低于多种子基线最差值（0.346），无需多种子验证。Phase 2 重测结果（0.561~0.589）在噪声带边界，但仍不如基线。

### EXP-05: max_bin 调整（M2）

| | |
|---|---|
| **问题** | LightGBM 默认 max_bin=255，是否调整分桶数能提升性能？ |
| **方法** | 测试 max_bin=63 和 max_bin=127。 |

**结果（seed=42 单种子）：**

| max_bin | Sharpe | vs seed=42 基线 |
|:-------:|:------:|:-------:|
| 255 (默认) | 0.609 | — |
| 127 | 0.370 | -39% |
| 63 | 0.254 | -58% |

**结论：** 模型仅 31 叶子节点，无法利用高分辨率分桶。降低 max_bin 丢失信息，增加也无益。max_bin=127 的 Sharpe 0.370（seed=42）在噪声带内（多种子基线 0.447±0.108），但未经多种子验证，结论可能因种子变化而改变。

## Phase 2: 组合构建层优化

### EXP-06: TopkDropout 策略

| | |
|---|---|
| **Commit** | `4267e1f` |
| **CLI** | `pangu backtest --top-n 30 --n-drop 10`（已成为默认值） |
| **问题** | 年换手率 77x，交易成本消耗 64% 的毛收益。Score 噪声导致排名剧烈波动。 |
| **方法** | Qlib TopkDropout：每次调仓只替换持仓中排名最差的 n_drop 只，填入排名最高的 n_drop 只非持仓股。组合大小恒定为 top_n。 |
| **原理** | 即使 score 有噪声，持仓股的排名大概率仍在合理范围内。只替换确实掉出范围的，避免因噪声导致的不必要换手。 |

**n_drop 参数探索：**

初始实现存在 bug（组合会缩小到 ~18.5 只），修复为 Qlib 标准"固定替换"后：
- 初步探索 n_drop = 0, 3, 5, 8, 10, 12, 15
- n_drop=10 为最优（在 top-30 组合中替换 1/3）
- n_drop=0（纯 Top-N 每周全量替换）换手率最高且 Sharpe 最低
- n_drop 过小（3-5）保留过多弱股；过大（12-15）换手过高

**最终结果（Qlib 标准实现，top-30, n_drop=10）：**

| 指标 | 无 Dropout (Phase 1) | TopkDropout (seed=42) |
|------|:-------------------:|:-----------:|
| Sharpe | 0.415 | **0.609** |
| Total Return | — | +46.7% |
| Excess Return | — | +53.0% |
| Annual Turnover | 77x | **34.2x** |
| Max Drawdown | — | -29.1% |

**多种子验证（5 seeds）：** TopkDropout 效果经配对 t 检验确认显著（p=0.013）。ndrop=0（纯 Top-N 全量替换）多种子结果：

| Seed | ndrop=0 Sharpe | ndrop=0 年化 | ndrop=0 MaxDD | ndrop=0 Turnover |
|:----:|:------:|:-------:|:-------:|:-------:|
| 42 | 0.274 | +3.9% | -32.7% | 65.4x |
| 123 | 0.168 | +1.7% | -34.0% | 65.3x |
| 456 | 0.165 | +1.6% | -36.2% | 65.6x |
| 789 | 0.297 | +4.2% | -35.2% | 65.2x |
| 2024 | 0.256 | +3.4% | -34.8% | 65.7x |
| **均值±标准差** | **0.232±0.062** | **+3.0%** | **-34.6%** | **65.4x** |

基线（ndrop=10）多种子均值 **0.447±0.108** vs ndrop=0 均值 **0.232±0.062**，Δ = +0.215，p = **0.013**。

**关键实现细节：**
- 初版有 bug：`s not in held` 应为 `s not in kept_set`（允许重新买入刚卖出的股票）
- 修复后从"keep zone"实现演化为 Qlib 标准"fixed swap"实现
- 从 Top-10 扩展到 Top-30，增加组合分散度

**结论：** TopkDropout 是整个优化过程中**最大的突破**，经多种子验证确认显著（p=0.013）：
- 多种子均值 Sharpe: 0.232（ndrop=0）→ 0.447（ndrop=10），**+93%**
- 换手率: 65.4x → 34.1x（-48%）
- 年交易成本大幅降低

### EXP-10: 训练集子采样（S4）

| | |
|---|---|
| **Commit** | `af35064` |
| **CLI** | `pangu train walkforward --train-subsample-stride 5` |
| **问题** | 5d 标签存在 80% 重叠（相邻天共享 4/5 的收益窗口），这种冗余是否影响模型？ |
| **方法** | 随机分块采样：将训练日期按 stride 大小分块，每块随机选 1 天（seed=42）。避免固定步长采样引入的时间效应（如星期效应）。仅对训练集采样，验证/测试集不变。 |
| **原理** | 业界标准做法（Qlib 推荐 `df.iloc[::k]`），减少重叠标签冗余。我们用随机分块替代固定步长，更通用且避免周期性偏差。 |

**结果（seed=42 单种子）：**

| 配置 | 样本量 | RankIC | Sharpe | vs seed=42 基线 |
|------|:-----:|:------:|:------:|:-------:|
| 5d 无采样 | ~330K | 0.066 | 0.609 | — |
| 5d stride=5 | ~66K (-80%) | 0.050 | 0.038 | -94% |
| 20d stride=10 | ~31K | 0.074 | 0.203 | -28% (vs 20d 基线) |

**失败分析：**
- 虽然标签高度相关，但相邻天的**特征**有微小差异（滚动窗口偏移 1 天）
- 这些微小差异为 GBM 提供了有用的分裂信息
- 子采样丢弃了这种特征多样性，导致严重欠拟合
- 5d stride=5 丢弃 80% 数据，效果灾难性

**结论：** 子采样对 GBM 无效。与 Deep Learning（可能受冗余样本梯度影响）不同，GBM 的 bagging/column sampling 已天然处理了冗余问题。5d stride=5 的 Sharpe 0.038 远低于多种子基线最差值（0.346），无需多种子验证。

## 种子稳健性与方法论

### 动机

GBM 在弱信号体制（IC~0.03）下对 `random_state` 高度敏感。单一种子的实验比较可能产生假阳性或假阴性。本节描述验证方法论和种子敏感度的量化分析。

### EXP-13: 种子敏感度分析

**方法**: 使用 5 个不同的 `random_state`（42, 123, 456, 789, 2024）分别训练完整 Walk-Forward 模型，独立回测。

**基线（默认配置，无时间衰减）结果**:

| Seed | Sharpe | 年化收益 | 最大回撤 |
|:----:|:------:|:-------:|:-------:|
| 42 | 0.609 | +10.5% | -29.1% |
| 123 | 0.475 | +7.7% | -29.7% |
| 456 | 0.346 | +5.0% | -34.7% |
| 789 | 0.350 | +5.2% | -34.6% |
| 2024 | 0.456 | +7.2% | -29.2% |
| **均值±标准差** | **0.447±0.108** | **+7.1%** | **-31.5%** |

- **seed=42 是 5 个种子中的最大值**（比均值高 36%），不能代表真实基线
- **变异系数 (CV)**: 24.2% — GBM 对初始化随机性极为敏感
- 真实基线 Sharpe 为 **0.447±0.108**，非此前报告的 0.609

### EXP-14: 随机选股 Bootstrap

**方法**: 保持 Top-30 等权、每周调仓结构不变，但不使用模型评分，而是从 ~800 只成分股中**随机选择** 30 只，重复 1000 次。

> ⚠️ 此 bootstrap 使用**简化回测**（无交易成本、无涨跌停处理），与完整回测引擎不直接可比。仅用于检验模型 vs 随机的统计显著性。

**结果（简化回测，无交易成本）**:

| | Sharpe |
|---|:---:|

## 核心发现

### 1. 种子敏感度与真实基线（EXP-13/15，最重要发现）

**GBM 在弱信号体制（IC~0.03）下对初始化随机性极为敏感。** 5 种子基线 Sharpe 均值为 **0.447±0.108**（CV=24%），seed=42 单独给出 0.609（高估 36%）。

**方法论启示**：所有实验结论都应基于多种子统计检验。单一种子的实验比较可能产生假阳性（如 EMA 平滑在单 seed 下被误判为"失败"，实际 p=1.000）或假阴性（如时间衰减在单 seed 下被低估，实际 p=0.023 显著）。

### 2. 时间衰减是唯一经统计验证的改进（EXP-07/15）

时间衰减（halflife=120 交易天）经 5 种子配对 t 检验确认显著（p=0.023）：
- Sharpe 均值 +41%（0.447→0.629）
- 种子间标准差 -60%（0.108→0.043）
- 年化收益从 +7.1% 提升到 +10.6%，最大回撤从 -31.5% 改善到 -26.0%

时间衰减不是简单的"偏好某些种子"，而是系统性地改善了信号质量。**已设为 CLI 默认配置**。

### 3. IC-Sharpe 悖论

**所有提升模型 IC/RankIC 的实验都降低了投资组合 Sharpe。** 这是贯穿全部实验的核心矛盾。

根因：Top-30 选股的边界信噪比 < 1.0。Grinold 基本法则 `IR = IC × √N`，其中 N=30（低 breadth）。提高全截面排名质量（IC）不能改善边界处的微小排名差异，反而因为模型更"自信"地在边界换股，增加了无效换手。

受影响实验: EXP-04 (早停放宽), EXP-08 (多周期融合), EXP-09 (多模型集成)

### 4. 组合层 >> 模型层（多种子验证确认）

| 优化层 | 多seed提升 | p-value | 代表实验 |
|--------|:-------:|:------:|---------|
| 组合构建层 | **+93%** | **0.013** | TopkDropout (ndrop0 0.232→ndrop10 0.447) |
| 信号层 | **+41%** | **0.023** | 时间衰减 (0.447→0.629) |
| 模型层 | +35% (一次性) | — | 超参调优 (seed=42 单种子) |
| 模型层 (后续) | n.s. | — | 早停变体 (seed=42 单种子) |

模型层在超参调优后即达到天花板。真正的收益来自组合构建层（TopkDropout）和信号层（时间衰减），且两者都经多种子验证确认。

### 5. 5d 标签是最优 Horizon

倒 U 型曲线与 Qlib 官方、学术共识完全一致。Alpha158 因子集（短周期技术因子为主）天然适配 5d 预测窗口。更长的 horizon（10d/20d）虽然 RankIC 更高，但标签自相关导致有效样本量急剧下降，Top-30 选股精度反而更差。（Horizon 对比为 seed=42 单种子数据。）

### 6. 信号层平滑无害也无益

多种子验证确认：Score EMA 平滑和重叠窗口集成**无效果**（p=1.000 和 p=0.238/0.840）。它们既不改善也不恶化结果，与 TopkDropout 功能重叠。

### 7. 子采样对 GBM 无效

虽然标签高度冗余（5d 相邻天重叠 80%），但 GBM 的 bagging（subsample=0.8）和列采样（colsample=0.7）已天然处理了冗余。相邻天的微小特征差异（滚动窗口偏移 1 天）为树分裂提供了有用信息，子采样反而丢弃了这种多样性。（seed=42 单种子，Sharpe 0.038 远离噪声带。）

### 逐实验业界对比

#### TopkDropout（EXP-06）✅ 与 Qlib 一致

**Qlib 官方实现**：TopkDropoutStrategy 是 Qlib 的默认组合构建策略，被广泛用于其 benchmark 和教程中（[源码](https://github.com/microsoft/qlib/blob/main/examples/portfolio/README.md)）。

**对比**：
- Qlib 默认 topk=50, n_drop=5（10% 替换率）
- PanGu 使用 topk=30, n_drop=10（33% 替换率）
- 两者都显示 TopkDropout 显著优于纯 Top-N 替换

**一致性**：**完全一致**。TopkDropout 是业界验证的有效方法。我们的更高替换率（33% vs 10%）反映了更短窗口下 score 波动更大的现实。

---

#### CSZScoreNorm 标签归一化（EXP-02）⚠️ 与 Qlib 看似矛盾

**Qlib 官方做法**：CSZScoreNorm 是 Alpha158 流水线的**默认标签处理器**（`_DEFAULT_LEARN_PROCESSORS` 中）。Qlib 文档称其"对防止模型偏向高波动时期至关重要"。学术文献（DeepWiki/MASTER 等）也推荐它作为截面建模标准。

**我们的结果**：Sharpe 从 0.308 暴跌到 0.045（-85%），模型偏好低波银行股。

**矛盾分析**：实际上**并不矛盾**，原因如下：
1. **评估方式不同**：Qlib 用 IC/RankIC 评估排名质量，CSZScoreNorm 确实提升排名（我们的 RankIC 未降）。但我们关注的是 **Top-N 多头组合的 Sharpe**，这取决于头部选股精度而非全截面排名。
2. **策略不同**：Qlib 的 benchmark 隐含 long-short 评估（IC 本身就是 long-short 概念）。CSZScoreNorm 对 long-short 有效（消除市场方向）。我们是纯多头 Top-N，需要模型识别**绝对**表现最好的股票，而非**相对**表现。
3. **数据量差异**：Qlib 7 年训练数据下 z-score 的截面分布稳定；18 个月下截面分布波动大，z-score 引入额外噪声。

**结论**：CSZScoreNorm 在 long-short + 长训练窗口场景下有效，在 Top-N 多头 + 短窗口场景下有害。两者不矛盾，是策略目标的差异。

---

#### LambdaRank 排序学习（EXP-03）⚠️ 学术有成功案例但条件严格

**学术文献**：MDPI 2022 论文 *"From Ranking Search Results to Managing Investment Portfolios"* 表明排序学习方法（含 LambdaRank）在组合管理中**可以**优于传统基线，但"只在精心的特征工程和合适评估下"。LightGBM 官方文档提醒 LGBMRanker 需要正确的 group 参数和稳定的 relevance 标签。

**我们的结果**：Mean RankIC = -0.043（完全逆向），灾难性失败。

**差异分析**：
1. **NDCG 设计初衷**：LambdaRank/NDCG 来自搜索引擎（query → 点击率），特点是 relevance 标签稳定、top-k 覆盖度高。金融领域 relevance（未来收益）极不稳定。
2. **弱信号体制**：IC~0.03 意味着信号极弱。NDCG 的指数加权放大了噪声，在 800 只股票中寻找 top-30 的梯度信号被淹没。
3. **A 股均值回归**：NDCG 过拟合到高波动动量股，而 A 股的短期动量有均值回归倾向，导致样本外系统性反转。

**结论**：LambdaRank 的学术成功案例通常使用更强的信号（IC>0.05）或更小的股票池。在弱信号 + 大宇宙场景下，它放大噪声而非提取信号。

#### 时间衰减样本加权（EXP-07）✅ 理论支持充分，多种子验证显著

**学术基础**：
- López de Prado, *Advances in Financial Machine Learning* (2018), Chapter 4: 明确讨论了时间衰减加权
- RiskLab AI ([文档](https://www.risklab.ai/research/financial-data-science/weights)): 推荐指数衰减加权处理金融数据的非平稳性
- Springer 2026 论文: "leak-safe return forecasting using time-decayed stacking"

**开源实现现状**：
- Qlib: **未内置**时间衰减（GitHub 源码搜索 0 结果）
- LightGBM: 原生支持 `sample_weight`，但没有预置衰减方案
- 其他主流框架（zipline, backtrader, FinRL）: 均未内置

**我们的结果**：多种子验证 Sharpe +41%（0.447→0.629），p=0.023（**统计显著**）。种子间标准差 -60%。

**分析**：理论界推荐但工业界不采用，可能原因：
1. Walk-Forward 本身就是"硬"时间衰减（每次只用最近 18 个月）
2. 增加一个难以调优的超参数（halflife），收益不确定
3. 效果高度依赖市场体制（我们验证了：熊市有效，牛市有害）

我们的多种子验证表明时间衰减确实有效，且大幅降低种子敏感度。PanGu 可能是首个提供多种子统计验证的开源实现。

---

#### 多周期标签融合（EXP-08/09）⚠️ 概念合理但实践困难

**学术文献**：
- Springer 2025, *MLSC: A Multi-label Stock Classifier for Multi-horizon Stock Trend*: 讨论了多周期预测的必要性，但承认"proper evaluation strategies are essential to avoid look-ahead bias"
- Qlib 官方使用单一 5d 标签作为默认 benchmark

**我们的结果**：所有融合方式（z-score/原始求和/多模型集成）均降低 Sharpe（-30%~-66%）。

**分析**：多周期融合在理论上合理（不同持仓周期的股票需要不同 horizon 信号），但实践中：
1. 长周期标签的自相关（20d lag-1 = 0.945）使有效样本量急剧下降
2. 融合等价于注入更多噪声（20d 模型的 Sharpe 仅 0.280，融入后拉低整体）
3. Qlib 坚持使用 5d 单一标签，说明他们也未解决这个问题

#### 训练集子采样（EXP-10）⚠️ 理论推荐 vs GBM 实际

**理论基础**：
- López de Prado (2018): 推荐对重叠标签进行子采样或使用 Triple-Barrier 事件化标签
- Qlib: 使用 `df.iloc[::k]` 风格的固定步长采样（非官方推荐，社区讨论）
- RiskLab AI: 强调重叠事件导致样本依赖性，建议非重叠采样

**我们的结果**：5d stride=5 Sharpe 暴跌（0.609→0.038，seed=42 单种子，-94%）。

**差异分析**：
1. **López de Prado 的语境**：他讨论子采样主要针对防止泄露和减少计算量，不是提升 GBM 性能
2. **GBM vs 深度学习**：GBM 的 bagging（subsample=0.8）+ column sampling（colsample=0.7）已天然处理冗余。深度学习的 SGD 梯度可能被冗余样本偏向，但 GBM 的逐叶分裂不受此影响
3. **特征多样性**：滚动窗口偏移 1 天产生的微小特征差异为 GBM 提供了有用的分裂候选。子采样丢弃了 80% 的这种多样性

**结论**：子采样对 SGD 类模型（Deep Learning）可能有效，但对 GBM 的 bagging 机制是冗余甚至有害的。

#### Score 时序平滑（EXP-12）⚪ 多种子验证确认无效果

**通用实践**：
- 信号平滑（EMA/SMA）是量化交易中的常见做法，用于减少噪声和假信号
- 通常应用于**价格基**信号（均线策略），而非 ML 模型输出
- 业界缺乏对 "ML score → EMA → TopkDropout" 这一特定流程的实证研究

**我们的结果**：多种子验证确认 EMA hl=1 **无效果**（p=1.000）。seed=42 下观察到的 -16% 完全是基线离群值造成。

**分析**：
- 传统 EMA 平滑针对的是直接交易信号（价格/动量），每个信号独立决定买卖
- ML score + TopkDropout 是**排名**系统，TopkDropout 已内置排名稳定机制（只替换最差的 n_drop 只）
- EMA 平滑与 TopkDropout 功能重叠，多种子验证证实既不增加也不减少价值

#### IC-Sharpe 悖论 ✅ 已知现象，我们提供了系统性实证

**学术和业界认知**：
- RevDog AI, *"Low IC, High Sharpe: The Quant Paradox Most Factor Models Get Wrong"*: 系统性讨论了 IC 和 Sharpe 的脱节
- Graham Capital, *"How Good is Good Enough?"*: 从数学角度解释了低 IC 可以产生高 Sharpe（通过分散化和低波动）
- Grinold 基本法则: IR = TC × IC × √BR，集中 Top-N 组合的 BR（breadth）极低，Transfer Coefficient 也低

**我们的贡献**：
- 通过 10+ 实验系统性验证了 IC↑→Sharpe↓ 的负相关
- 确认了在 IC~0.03 的弱信号体制 + Top-30 集中组合下，这个悖论尤其严重
- 量化了具体数值：M1a 提升 RankIC 从 0.066→0.069（+4.5%），但 Sharpe 从 0.609→0.589（-3.3%，seed=42 单种子）

### 总结：与业界的一致性

| 实验 | 与业界一致？ | 说明 |
|------|:----------:|------|
| TopkDropout | ✅ 完全一致 | Qlib 标准策略，广泛验证有效，多种子确认 p=0.013 |
| CSZScoreNorm | ⚠️ 表面矛盾 | Qlib 用于 long-short + 7 年数据；我们是 Top-N + 18 月，差异合理 |
| LambdaRank | ⚠️ 条件性一致 | 学术有成功案例但需强信号；我们的弱信号体制下失败符合预期 |
| 时间衰减 | ✅ 理论一致 | 学术推荐但开源不采用，我们多种子验证确认显著（p=0.023） |
| 5d 最优 Horizon | ✅ 完全一致 | Qlib 默认 5d，学术共识 3-10d sweet spot |
| 多周期融合 | ⚠️ 理论合理但实践失败 | 学术讨论多但无成功的开源实现 |
| 子采样 | ⚠️ 条件性不一致 | López de Prado 推荐但针对不同目的（防泄露 vs 提升 GBM）|
| Score 平滑 | ⚪ 无效果 | 多种子验证确认既不有害也不有益（p=1.000）|
| IC-Sharpe 悖论 | ✅ 已知现象 | 我们提供了系统性实证数据 |

**核心差异根源**：PanGu 与 Qlib 的最大区别在于**数据量和训练方式**。Qlib 用 7 年数据一次训练，IC~0.045；PanGu 用 18 月滚动训练，IC~0.036。这个信号强度差异导致了许多方法在两个系统上效果不同。在弱信号体制下，模型层优化几乎无效，组合层优化（TopkDropout，p=0.013）和信号层优化（时间衰减，p=0.023）成为唯一经统计验证的收益来源。
